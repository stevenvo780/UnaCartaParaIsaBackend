    Class SimulationRunner

    Main simulation orchestrator and coordinator.

    Manages the game state, coordinates all simulation systems, handles commands, and provides snapshots for client synchronization. Uses MultiRateScheduler for optimized system updates at different frequencies.

    See

    • MultiRateScheduler for update rate management
    • StateCache for snapshot optimization
    • DeltaEncoder for delta snapshot generation

    initialize

    • initialize(): Promise<void>

      Initializes the simulation runner and all dependent systems.

      Performs the following setup operations:

      • Initializes GPU compute service (with CPU fallback)
      • Rebuilds entity and spatial indices
      • Configures inter-system dependencies (lifecycle, needs, AI, economy, etc.)
      • Sets up event listeners for cross-system communication
      • Creates initial family (Isa, Stev, and 4 children) if no agents exist
      • Creates initial infrastructure (house, workbench, storage, rest, kitchen)
      • Initializes movement states for all agents
      • Registers all systems in the multi-rate scheduler

      Returns Promise<void>

      Throws

      If GPU initialization fails (falls back to CPU)

      • Defined in src/domain/simulation/core/SimulationRunner.ts:375

    saveSimulation

    • saveSimulation(): Promise<void>

      Saves the current simulation state to storage.

      Returns Promise<void>

      Remarks

      Clones the authoritative state before persisting it asynchronously via the storage service. Callers may await the returned promise for logging purposes, but the simulation loop can safely fire-and-forget without race conditions because mutation occurs on the cloned snapshot.
